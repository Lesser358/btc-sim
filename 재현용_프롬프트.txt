[차트 기능]
─────────────────────────────────────────
인터벌 버튼 15개:
  1분 | 3분 | 5분 | 15분 | 30분 | 1시간 | 2시간 | 4시간 | 6시간 | 8시간 | 12시간 | 1일 | 3일 | 1주 | 1월
  Binance API 키: 1m/3m/5m/15m/30m/1h/2h/4h/6h/8h/12h/1d/3d/1w/1M

초기 로드: 500봉, 최근 150봉만 보이도록 setVisibleLogicalRange 설정 (fitContent 사용 금지)

Hull MA (HMA):
  - ON/OFF 토글 스위치
  - 기간 입력란 (기본 55, 2~400)
  - 색상: #f0b90b (노랑)
  - 알고리즘:
    half = floor(period/2), sq = floor(sqrt(period))
    WMA(half) × 2 − WMA(period) 계산 후 → 그 결과값에 WMA(sq) 적용
    null 구간(period-1 이전)은 0으로 치환 금지 → 유효 데이터만 분리해서 계산

차트 수평선 (포지션 보유 시 자동 표시):
  - 진입가: 실선 (롱=초록, 숏=빨강)
  - 청산가: 점선 (노랑)
  - TP 가격: 점선 (초록)
  - SL 가격: 점선 (빨강)

차트 마커:
  - 진입: 화살표 (롱=아래서 위 초록, 숏=위서 아래 빨강)
  - 종료: 회색 원
  - 강제청산: 노랑 원
  - 최근 30개 내역 기준

[거래 로직]
─────────────────────────────────────────
초기 잔고: 5,000 USDT

수수료:
  - 시장가(Taker): 0.05%
  - 지정가(Maker): 0.02%

포지션:
  - 격리 마진 / 교차 마진 전환 가능 (포지션 보유 중 전환 불가)
  - 레버리지 2~125배
  - 롱/숏 동시 보유 불가 (반대 방향 주문 시 기존 포지션 감소/종료/전환)
  - 부분청산: 1~99% 자유 입력

TP/SL:
  - 가격 직접 입력 모드
  - % 입력 모드: 진입가 기준 수익률로 실제 가격 자동 계산
  - 체결 시 pending 배열에 {kind:"tp"|"sl", side, qty, price} 형태로 저장
  - 매 틱마다 체크, 조건 충족 시 closePosition(1) 호출

지정가 주문:
  - pending 배열에 저장, 매 틱마다 가격 체크
  - 롱 지정가: lastPrice <= 주문가일 때 체결
  - 숏 지정가: lastPrice >= 주문가일 때 체결

청산가 계산 (바이낸스 BRACKETS 방식):
  - 심볼별 유지증거금 구간(mmr, cum) 적용
  - 격리: margin = 투입한 증거금
  - 교차: margin = walletBalance 전체
  - 롱 청산가: (qty×entry - margin - cum) / (qty×(1 - mmr))
  - 숏 청산가: (qty×entry + margin + cum) / (qty×(1 + mmr))

강제청산 조건:
  - 롱: markPrice <= 청산가
  - 숏: markPrice >= 청산가
  - 청산 시 격리모드: 투입 증거금 소멸 / 교차모드: 지갑 잔고 전체 소멸

펀딩비:
  - 8시간마다 (바이낸스 nextFundingTime 기준)
  - 롱: 지불 = notional × fundingRate
  - 숏: 수령 = notional × fundingRate
  - 중복 정산 방지: lastFundingT !== nextFundingT 체크
