# AI 분석 기능 검증 체크리스트

목표: `🤖 AI 분석` 탭이 실제 거래 데이터 상태별로 깨지지 않고, 잘못된 투자 조언이나 보안 위험 없이 동작하는지 확인한다.

## 검증 기록

- 2026-06-23 1차 검증 완료: 테스트 전용 origin(`127.0.0.1:7824`)에서 비로그인 모드로 확인.
- 발견/수정: 실현손익 거래 3~4회에서도 데이터 부족 화면을 건너뛰던 기준 불일치 수정(`st.total < 3` → `st.total < 5`).
- 확인 결과: 0회 데이터 부족, 6회 거래 분석 화면, 동의/철회/재동의, 모바일 375px, 포지션 보유 모바일 폭발 회귀, 콘솔 에러 0 통과.

## 필수 검증

- [x] 거래 내역 0건: 데이터 부족 화면 표시, 콘솔 에러 0
- [x] 거래 내역 1~4건: 필요한 추가 거래 수 표시, 콘솔 에러 0
- [x] 거래 내역 5건 이상: DNA, 수익/손실 원인, 추천, 핵심 통계 표시
- [ ] 수익 계좌: 수익 원인이 승률/손익비/레버리지 기준으로 자연스럽게 표시
- [x] 손실 계좌: 손실 원인이 손익비/청산/방향 편향 기준으로 자연스럽게 표시
- [ ] 강제청산 포함 계좌: 청산 경고와 SL 권장 문구 표시
- [ ] 롱 편향 계좌: 방향 편향 경고 또는 롱 전문 유형 표시
- [ ] 숏 편향 계좌: 방향 편향 경고 또는 숏 전문 유형 표시
- [ ] 레버리지 과다 계좌: 레버리지 축소 추천 표시
- [x] 분봉/종목별 승률 데이터가 있는 계좌: 최적 분봉/코인 추천 표시

## 동의 흐름

- [x] 최초 진입: AI 활용 동의 화면 표시
- [x] 동의 클릭: 분석 화면 진입
- [x] 거절 클릭: 거절 상태 화면 표시
- [x] 다시 동의하기: 분석 화면 재진입
- [x] 동의 철회: 동의 전 상태로 복귀
- [ ] 새로고침 후: 동의 상태가 localStorage 기준으로 유지

## 모바일 검증

- [x] 375px 폭에서 AI 분석 탭 레이아웃 깨짐 없음
- [x] 카드/통계/추천 문구가 화면 밖으로 넘치지 않음
- [x] `document.body.scrollWidth === document.documentElement.clientWidth`
- [x] 하단 탭 전환 시 기존 포지션/미체결/내역 탭 레이아웃 회귀 없음
